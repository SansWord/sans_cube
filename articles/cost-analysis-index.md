## How to add a new feature's cost analysis

When a new feature ships (workflow: design → plan → implement → review across 4 sessions named `{FEATURE_NAME} - design`, `{FEATURE_NAME} - plan`, `{FEATURE_NAME} - implement`, `{FEATURE_NAME} - review`):

1. Run `scripts/cost_extract.py --label {phase}` for each of the 4 sessions.
2. Create `articles/cost-{feature-slug}.md` in the format used by the existing per-feature files (title, **Shipped:** line, per-phase tables, summary table, notes).
3. Add a row at the top of the [Per-feature breakdowns](#per-feature-breakdowns) table above.
4. Append a dated entry to the [Data points](cost-comparison-model-strategy.md#data-points) section of `cost-comparison-model-strategy.md`. Note what the new point **confirmed** and what it **changed**.
5. If the new evidence shifts the recommendation, update the **"Current Recommendation"** block at the top of `cost-comparison-model-strategy.md` (including the date and N).
6. If the new evidence doesn't change recommendations, say so explicitly in the data-point entry.

The "ship it" shortcut in the project CLAUDE.md covers code/docs/devlog. The **"add cost analysis of this feature"** shortcut (also in CLAUDE.md) covers this index and the living document.
